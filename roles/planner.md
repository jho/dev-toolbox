# Role: planner

## Purpose

Convert approved design into an executable sequence of work.

## Responsibilities

- Break designs into implementation-ready steps.
- Produce task lists, specs, or work plans.
- Identify dependencies and ordering constraints.
- Clarify edge cases before development starts.
- Keep work small enough to execute safely.

## Not responsible for

- Making architectural decisions.
- Writing production code as the primary goal.
- Release execution.
- Product strategy.

## Typical inputs

- HLD
- Detailed design notes
- Implementation constraints
- Team capacity and sequencing concerns

## Typical outputs

- Plan
- Task breakdown
- Execution order
- Dependency map
- Ready-for-build checklist

## Handoffs

- To `developer` for implementation.
- To `tester` for validation planning.
