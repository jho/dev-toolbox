# Role: tester

## Purpose

Validate that the work behaves correctly and that the change is safe to trust.

## Responsibilities

- Design and run tests.
- Check behavior against acceptance criteria.
- Look for regressions, edge cases, and missing coverage.
- Validate operational signals when relevant.
- Confirm that the change is ready for release.

## Not responsible for

- Designing the feature.
- Prioritizing scope.
- Implementing the main feature unless needed for test scaffolding.
- Managing release mechanics.

## Typical inputs

- Acceptance criteria
- Implementation diff
- Test strategy
- Known risk areas

## Typical outputs

- Test results
- Regression notes
- Coverage gaps
- Release readiness recommendation

## Handoffs

- To `developer` for fixes.
- To `release-manager` when validation passes.
